## SaaS & Commercial Products

:::caution[Short Answer: YES, It's Free]
**A commercial organization CAN use unmodified Fluxbase as their product's backend without any license fee.** AGPLv3 only requires source disclosure if you **modify** Fluxbase's source code.
:::

:::tip[The Rule]
**Using ≠ Modifying.** Using Fluxbase as your backend (unmodified) is always free, even for commercial products. You only need a commercial license if you change Fluxbase's Go source code and want to keep those changes private.
:::

Your application code is **separate** from Fluxbase and can be under any license you choose. AGPLv3 only applies to Fluxbase and modifications to Fluxbase—not your application.

| SaaS Scenario                                | License Needed |
| -------------------------------------------- | -------------- |
| SaaS using Fluxbase as backend (unmodified)  | **Free**       |
| SaaS with custom edge functions (your code)  | **Free**       |
| SaaS with modified Fluxbase, shared publicly | **Free**       |
| SaaS with modified Fluxbase, kept private    | **Commercial** |

## Frequently Asked Questions

### My company wants to use Fluxbase as our product's backend. Is this free?

**Yes, absolutely.** A commercial organization can use unmodified Fluxbase as their product's backend **without any license fee**. This includes:

- SaaS products
- Mobile app backends
- Enterprise applications
- Internal company tools
- Any commercial product

You only need a commercial license if you **modify Fluxbase's Go source code** and want to keep those modifications private.

### I'm building a SaaS product. Do I need a license?

**Probably not!** You only need a commercial license if you:

1. **Modify Fluxbase source code** (not your app code), AND
2. **Want to keep those modifications private**

Most SaaS products use Fluxbase as-is for:

- Authentication
- Database API
- Realtime subscriptions
- File storage
- Edge functions

All of this is **free** under AGPLv3. Your SaaS application code, business logic, and edge functions are separate from Fluxbase and can be under any license.

### What counts as "modifying Fluxbase"?

| ✅ NOT a modification       | ❌ IS a modification        |
| --------------------------- | --------------------------- |
| Writing edge functions      | Changing Go source code     |
| Configuring settings        | Patching core functionality |
| Creating database schemas   | Modifying internal modules  |
| Using the REST API          | Forking and customizing     |
| Adding authentication rules | Extending built-in services |

### What if I don't modify Fluxbase at all?

You can use Fluxbase for **any purpose**—including commercial SaaS products—without a license. AGPLv3 obligations only trigger when you modify Fluxbase itself.

### How does AGPLv3 affect SaaS?

AGPLv3's "network copyleft" provision means that if you modify Fluxbase and offer it as a network service (like SaaS), you must make your modified source code available to users. A commercial license removes this requirement.

### Can I use Fluxbase for internal company tools?

Yes! Internal tools are free under AGPLv3. If you modify Fluxbase, you only need to share modifications if you distribute the software outside your organization. Internal deployment doesn't trigger distribution requirements.

- **Indemnification**: Legal protection available (on request)
